### Task 1 — `ports` table schema + migration
**Agent: main thread**

Add to `apps/server/src/db/schema.ts` using the `sqliteTable` pattern:

```ts
export const ports = sqliteTable('ports', {
  id: integer('id').primaryKey({ autoIncrement: true }),
  sandboxId: text('sandbox_id').notNull(),
  port: integer('port').notNull(),
  public: integer('public', { mode: 'boolean' }).notNull().default(false),
  createdAt: integer('created_at', { mode: 'timestamp_ms' }).notNull().$defaultFn(() => new Date()),
  updatedAt: integer('updated_at', { mode: 'timestamp_ms' }).notNull().$defaultFn(() => new Date()),
}, (t) => ({
  sandboxPortUniq: uniqueIndex('ports_sandbox_port_uniq').on(t.sandboxId, t.port),
}));

export type PortRow = typeof ports.$inferSelect;
export type PortInsert = typeof ports.$inferInsert;
```

Run `pnpm db:generate` to produce `apps/server/drizzle/0006_*.sql`. Commit both files.

### Task 2 — Preview proxy (parallel with Task 3)
**Agent: implementer (opus)**

#### `apps/server/src/proxy/host.ts`
Export `parsePreviewHost(host: string, previewDomain: string): { name: string; port: number } | null`.
- Strip optional `:port` suffix from `host` first.
- Case-insensitive match: `host` must equal `<label>.<previewDomain>` exactly.
- `label` must match `/^([a-z0-9][a-z0-9-]*)-(\d{2,5})$/`. The last hyphen-delimited numeric segment is the port; everything before is the sandbox name.
- Port must be in range 1–65535. Return null for any mismatch.
- No imports of db, fastify, or I/O modules. Pure function.

#### `apps/server/src/proxy/ports.repo.ts`
- `listPorts(db: Db, sandboxId: string): Promise<PortRow[]>`
- `getPort(db: Db, sandboxId: string, port: number): Promise<PortRow | undefined>`
- `createPort(db: Db, insert: PortInsert): Promise<PortRow>` — throws on unique-constraint violation (caller catches for 409)
- `setPublic(db: Db, sandboxId: string, port: number, value: boolean): Promise<void>` — updates `updatedAt`
- `deletePort(db: Db, sandboxId: string, port: number): Promise<void>`
- `appendPortEvent(db: Db, sandboxId: string, port: number, kind: 'port.exposed' | 'port.hidden'): Promise<void>` — inserts into `events` table, payload `{port}`, consistent with `appendSandboxEvent` pattern

#### `apps/server/src/proxy/routes.ts`
Export `registerPortRoutes(app: FastifyInstance, deps: { db: Db; previewDomain?: string })`. All routes session-gated by the global auth gate (no extra auth code needed). Validate `:id` exists in sandboxes table before port operations (404 if absent).

- `GET /api/sandboxes/:id/ports` → `{ ports: PortInfo[] }`. `url` field is `null` when `previewDomain` is absent, otherwise `https://<name>-<port>.<previewDomain>`.
- `POST /api/sandboxes/:id/ports` body `{ port: number }` (zod, integer, min 1, max 65535). Reject port in `DENIED_PORTS = [8022]` with 422 `{ error: 'port not allowed' }`. On duplicate unique constraint, return 409 `{ error: 'port already registered' }`. On success 201.
- `PATCH /api/sandboxes/:id/ports/:port` body `{ public: boolean }` (zod). Toggle; append `port.exposed` / `port.hidden` event via `appendPortEvent`. 200 on success, 404 if row absent.
- `DELETE /api/sandboxes/:id/ports/:port` → 204. 404 if row absent.

#### `apps/server/src/proxy/proxy.ts`
Export `registerPreviewProxy(app: FastifyInstance, deps: { db: Db; previewDomain: string })`.

Register an `onRequest` hook at the top of the hook chain (before auth gate). If `req.headers.host` does not match the preview domain pattern via `parsePreviewHost`, call `done()` and fall through untouched.

When a preview request is matched:

**Security invariants (enforce all five; each is a rejection criterion if missing):**
1. Resolve target exclusively from DB: look up sandbox by parsed `name`, must exist with `status = 'running'`. Derive forward address as `<machineId>.vm.<app>.internal:<port>` (same pattern as `terminalEndpoint` in fly-provider) or `privateIp`. Never use any value from Host, path, or query for the target address. Run `assertTargetShape(hostname)` guard (validates the resolved hostname matches internal-DNS or private-IP shape; unit-tested separately) before dialing.
2. Port must exist as a row in `ports` for that sandbox. No row → 404 HTML response. Additionally, `DENIED_PORTS = [8022]` is hard-checked at proxy layer even if a row somehow exists (defense in depth).
3. Auth: if `port.public = false`, call `authorizeToken(db, tokenFromRequest(req))` (from `apps/server/src/auth/middleware.ts`). Return 401 minimal HTML when token absent/invalid. If `port.public = true`, skip auth.
4. `assertTargetShape` guard: assert hostname matches `/^[a-z0-9-]+\.vm\.[a-z0-9-]+\.internal$/` or is a private RFC-1918 IP. Throw (→ 502) on any other shape. This structurally prevents proxying to localhost, 127.0.0.1, the collector, or the DB.
5. Strip the `outpost_session` cookie from the forwarded request in ALL cases (both public and private). Forward all other request headers. Set `x-forwarded-host` and `x-forwarded-proto` on the forwarded request. Do not forward hop-by-hop headers (`connection`, `keep-alive`, `transfer-encoding`, `upgrade` on HTTP/1.1 non-upgrade paths).

**HTTP forwarding:** Use `undici` (already a Fastify transitive dep; zero extra install cost) for streaming forward. Stream response body directly — no buffering. Propagate status code and response headers minus hop-by-hop. On upstream connect error return 502.

**WebSocket passthrough:** Handle via `server.on('upgrade', ...)` or equivalent. Apply the same host-check, sandbox/port/auth invariants. Then open a plain WS connection to the target and pipe both directions with close/error propagation. Forward `sec-websocket-protocol` unmodified. Do not modify subprotocols (HMR compatibility).

**Proxy disabled path:** When `previewDomain` is absent, `registerPreviewProxy` is not called. Preview-host requests hit normal Fastify routing and 404 naturally.

#### Tests (`apps/server/src/proxy/__tests__/`)

`host.test.ts` — table-driven, all cases:
- Valid: `my-sandbox-3000.sandbox.outpost.dev` → `{ name: 'my-sandbox', port: 3000 }`
- Valid: label with internal hyphens `my-cool-box-8080.sandbox.outpost.dev` → `{ name: 'my-cool-box', port: 8080 }`
- Valid: Host header with `:443` suffix stripped first
- Case-insensitive match
- Wrong domain → null
- Spoofed suffix `evil-sandbox.outpost.dev.attacker.com` → null
- Bad label start char `9bad-3000.sandbox.outpost.dev` → null
- Port below range `my-box-0.sandbox.outpost.dev` → null
- Port above range `my-box-99999.sandbox.outpost.dev` → null
- Bare domain (no label) → null
- Wildcard attempt `*.sandbox.outpost.dev` → null

`proxy.test.ts`:
- Auth gate: private port + no cookie → 401 HTML; private port + valid session cookie → 200 forwarded
- Public port + no cookie → 200 forwarded
- No `ports` row for sandbox → 404
- Port 8022 row somehow exists → proxy hard-denies, 404
- Cookie stripping: assert `outpost_session` not present in upstream-received headers (test with a fake HTTP server that echoes received headers)
- WS passthrough echo: connect WS to fake upstream WS server (startFakeDaemon pattern), send frame, assert echo received
- `assertTargetShape` unit: valid internal DNS → passes; `localhost` → throws; `127.0.0.1` → throws; `10.0.0.1` (RFC-1918) → passes
- Proxy disabled (no `previewDomain` env) → hook absent, request falls through to 404

## Risks & gotchas
- Wildcard TLS + DNS (`*.<previewDomain>`) requires Fly cert setup — not automatic; document steps in known-limitations, do not assume.
- Local dev has no wildcard DNS: `parsePreviewHost` is unit-tested with injected Host headers; proxy integration tests use fake upstream HTTP/WS servers, not a live Fly sandbox.
- SSRF via crafted subdomain or spoofed Host — mitigated by invariants 1–4: target from DB only, `assertTargetShape` guard before dial, `DENIED_PORTS` constant.
- HMR WebSocket compatibility: do not touch `sec-websocket-protocol`; passthrough test covers a basic echo but real Vite HMR verification requires a live sandbox — fold into live-Fly verification task.
- `events.payload` is JSON text — parse with try/catch in UI; fall back to raw `kind` string, never throw.
- `public` toggle defaults `false`; flipping to `true` requires both UI confirm step and explicit PATCH — both layers must enforce this independently.
- `undici` is a Fastify transitive dep (brought in by `fastify` itself); no new package entry needed — verify with `pnpm ls undici` before implementer uses it.
- `proModules.has(...)` check must exist in `SandboxList.tsx` source even though it always returns false — this is the open-core seam anchor, not dead code.
