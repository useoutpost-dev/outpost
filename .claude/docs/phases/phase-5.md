# Phase 5 — Telemetry & Usage (Lookout)

Route OTLP usage metrics from sandboxes into SQLite and surface token/cost totals in the UI.

## Task breakdown

### Task 1 — Schema + migration
**Agent: main thread**

Add the `usage` table to `apps/server/src/db/schema.ts` using the `sqliteTable` pattern already present in that file:

```ts
export const usage = sqliteTable('usage', {
  id: integer('id').primaryKey({ autoIncrement: true }),
  ts: integer('ts', { mode: 'timestamp_ms' }).notNull().$defaultFn(() => new Date()),
  sandboxId: text('sandbox_id').notNull(),
  accountId: text('account_id'),          // nullable — auth mode "none"
  model: text('model').notNull(),
  inputTokens: integer('input_tokens').notNull().default(0),
  outputTokens: integer('output_tokens').notNull().default(0),
  cacheReadTokens: integer('cache_read_tokens').notNull().default(0),
  cacheWriteTokens: integer('cache_write_tokens').notNull().default(0),
  estCostUsd: real('est_cost_usd').notNull().default(0),
}, (t) => ({
  sandboxTsIdx: index('usage_sandbox_ts_idx').on(t.sandboxId, t.ts),
  tsIdx: index('usage_ts_idx').on(t.ts),
}));

export type UsageRow = typeof usage.$inferSelect;
export type UsageInsert = typeof usage.$inferInsert;
```

Run `pnpm db:generate` to produce `apps/server/drizzle/0005_*.sql`. Commit both files.

### Task 2 — Collector + normalize (parallel with task 3)
**Agent: implementer (opus)**

**`loadBootConfig` changes (`apps/server/src/index.ts`):**
- Read `OUTPOST_COLLECTOR_TOKEN` from env; throw if absent or shorter than 32 chars (same fail-loud style as `OUTPOST_MASTER_KEY`). Error message must not echo the token value.
- Add `collectorToken: string` to `BootConfig.sandbox`. Propagate through the boot sequence to `buildApp` opts and on to `registerCollectorRoutes`.

**`PUBLIC_PATHS` change (`apps/server/src/auth/middleware.ts`):**
- Add `'/v1/metrics'` to the `PUBLIC_PATHS` set. The collector is token-gated rather than session-gated.

**`apps/server/src/telemetry/collector.ts`:**
- Export `registerCollectorRoutes(app: FastifyInstance, deps: { db: Db; collectorToken: string })`.
- Register `POST /v1/metrics`. Set a body size limit (e.g. `bodyLimit: 1_048_576` — 1 MB) on the route or plugin scope.
- Auth check: extract `Authorization` header, verify it is `Bearer <collectorToken>` using a constant-time comparison (`crypto.timingSafeEqual`). Return 401 on mismatch, before any body parsing.
- On success: call `normalize(body)` from `normalize.ts`. If the result array is empty (malformed/unrecognized), return 400 with `{ error: 'unrecognized payload' }`. Otherwise call `insertUsageRows(db, rows)` and return 200 `{ accepted: rows.length }`.
- Never throw uncaught: wrap in try/catch, return 400 for parse errors, log server errors and return 500.

**`apps/server/src/telemetry/normalize.ts`:**
- Export `normalize(body: unknown): UsageInsert[]`.
- Parse the OTLP/HTTP JSON structure: `resourceMetrics[].resource.attributes[]` for `sandbox.id` and `account.id`; `resourceMetrics[].scopeMetrics[].metrics[]` for metric data points.
- Recognized metric names: `claude_code.token.usage` (attribute `type` in `{input, output, cacheRead, cacheCreation}`) and `claude_code.cost.usage`. Group by `(sandboxId, model)` within a single payload before inserting so one OTLP batch becomes one row per sandbox+model pair.
- Allowed output fields per row: `sandboxId`, `accountId`, `model`, `inputTokens`, `outputTokens`, `cacheReadTokens`, `cacheWriteTokens`, `estCostUsd`. No other fields. Attribute keys not in this list are silently dropped before any object is constructed.
- On any parse error (wrong shape, missing required field) return `[]` — never throw.
- This is a pure function with no imports of db, fastify, or any I/O module.

**`apps/server/src/sandboxes/service.ts`:**
- Extend the sandbox env injection block (lines ~163–171) with:
  - `OTEL_EXPORTER_OTLP_PROTOCOL=http/json`
  - `OTEL_EXPORTER_OTLP_HEADERS=Authorization=Bearer <collectorToken>`
- The `collectorToken` value must flow from `config` (already typed in `SandboxBootConfig`-equivalent shape); do not read `process.env` directly in the service.

**Tests (`apps/server/src/telemetry/__tests__/`):**
- `normalize.test.ts`: fixture `otlp-valid.json` (recorded real-shape payload) → expected `UsageInsert[]`; fixture `otlp-malformed.json` → `[]`. Assert output keys are exactly the allowed set using `Object.keys(row).sort()`.
- `collector.test.ts`: POST without header → 401; POST with wrong token → 401; POST with correct token + valid body → 200 + one `db.insert` call (assert via spy or DB row count, not per-datapoint); POST with correct token + malformed body → 400.
- No-prompt-content grep assertion: inline test using `fs.readFileSync` on `normalize.ts` source, assert the string `'prompt'` and `'content'` do not appear (case-insensitive).

## Risks & gotchas
- OTLP payload shape churns across Claude Code versions — normalize fixtures pin a specific payload shape; when the shape changes only `normalize.ts` and its fixtures need updating, nothing else.
- Collector must never be publicly reachable — defense in depth: token auth AND Fly private network. Document reliance on network scoping in known-limitations; do not remove token auth even if the endpoint is private.
- `OTEL_EXPORTER_OTLP_HEADERS` value contains the collector token as machine env — same known-limitations category as `OUTPOST_MASTER_KEY`; acceptable, must be documented in the Phase 4 known-limitations file.
- OTLP protocol mismatch: Claude Code may default to grpc/protobuf; `OTEL_EXPORTER_OTLP_PROTOCOL=http/json` forces HTTP/JSON to keep the collector dependency-free. Verify metric delivery with a live Fly hand-test (same bucket as the Phase 4 live login test).
- SQLite write volume from metrics — batch inserts (`db.insert(usage).values([...]).run()`) are mandatory; per-event writes are a rejection criterion.
- "est. API value" label and "estimated" in `UsageBar` are hard requirements — incorrect copy is a rejection criterion even if the code is otherwise correct.
- Per-sandbox scoping missing from a query is a privacy bug even in single-user Tier 0 — any unscoped aggregate in `usage.repo.ts` is a rejection criterion.
